Layout Breakdown

  • Header: Recording indicator (animated red dot + "REC"), session timer in large tabular-nums font, tag count badge
  • Waveform: Compact live waveform (~48pt) below timer — visual heartbeat confirming recording is active, subtle enough to not distract
  • Category Tabs: Segmented control with color dots — tap to filter tags by category. Active category highlighted
  • Tag Grid: 2-column card layout with color-coded left stripe. Each card is a large tap target (minimum 48pt height)
  • Custom + Stop: Dashed "+" card for custom tags, solid bar for stop recording

Timestamp-First Interaction

Every tag tap immediately captures the rewind-anchor timestamp. The moment is locked in on first touch — categorization is secondary. For the "+" custom tag, tapping captures the timestamp first, then presents a quick label input. No moment is ever lost waiting for the DM to type.

Waveform Purpose

The waveform isn't for navigation (that's the Mac's job). It's a confidence signal — visual proof the recording is alive. Especially valuable 2 hours into a session when the DM glances at the phone and needs instant reassurance without reading text.

Why Timestamp-First Matters

Traditional approach: DM taps "+" → types a label → then the timestamp is captured. Problem: by the time they finish typing, the rewind-anchor is 10 seconds late — it captures the typing, not the moment.

Timestamp-first approach: The tap on "+" immediately locks the rewind-anchor. The label input comes after, but the audio reference points to the correct moment. This decouples "when" from "what" — the time-critical capture from the non-critical categorization.

Detail Area Breakdown

  • Tag header: Editable label + category badge + timestamp (shows both tag time and rewind-anchor start)
  • Transcription: WhisperX output, editable inline — click to correct fantasy names or garbled text
  • Notes: Free-form text area for DM annotations — context that audio alone can't capture
  • Actions: Edit label, change category, delete. Inline, not buried in menus
  • Related tags (right): Cross-session search results for the current tag's content — surfaces connections automatically. "Other mentions of Grimthor" across all sessions
  • Session summary (right): Session-level notes visible alongside tag detail for context

Top-to-Bottom Review Flow

The DM's eyes move naturally: sidebar (pick a tag) → waveform (see where it falls in the session) → playback (hear it) → transcription (read it) → notes (annotate it). No panel-switching, no mode changes. The entire review workflow is a single vertical scroll.

Final Design Direction Summary

The chosen direction combines D&D Beyond-inspired organization with Headspace's calm confidence, connected by a timestamp-first interaction model that never loses a moment.

iOS Recording Screen

  • Layout: Header + waveform + category tabs + card grid
  • Waveform: Compact live visualization for recording confidence
  • Tags: 2-column card grid with category tabs and color stripes
  • Interaction: Timestamp-first — every tap instantly captures the rewind-anchor
  • Custom tags: "+" captures timestamp on first tap, categorize/label as second step
  • Feedback: Haptic on every tag, visual pulse, count increment

Mac Review Screen

  • Layout: Tag sidebar (left) + waveform (center-top) + detail (center-bottom)
  • No right panel: Detail area lives below waveform, giving it full width
  • Tag selected: Transcription + notes + actions appear below waveform
  • Related tags: Cross-session search results shown alongside current tag
  • Review flow: Top-to-bottom — see, hear, read, annotate
  • Session notes: Summary visible in context, not a separate screen

Key Innovation: Timestamp-First

  • Problem: Traditional bookmarks capture the moment you finish categorizing, not the moment you noticed
  • Solution: Every tap immediately locks the rewind-anchor timestamp
  • Effect: Categorization becomes non-time-critical — DM can take 5 seconds to pick a label without losing the moment
  • Applies to: Both standard tags (instant) and custom "+" tags (timestamp on first tap)

Design Principles Applied

  • Table presence first: Large tap targets, haptic feedback, waveform confidence signal
  • Capture fast, review rich: iOS is organized but minimal; Mac is information-dense with inline editing
  • Archive earns value: Cross-session related tags surface connections automatically
  • Trust through reliability: Waveform proves recording is active; timestamp-first ensures no moment is lost
